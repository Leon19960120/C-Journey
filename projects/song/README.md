# 🎵 C语言 歌词同步音乐播放器 (Music Player)

这是一个基于 C 语言开发的终端音乐播放器项目。项目通过调用系统底层播放器（如 `mplayer`）实现音频播放，并结合多线程或时间轮询机制，在终端（Console）上实现歌词（.lrc 文件）与音乐的精准实时同步滚动显示。

---

## 📂 项目目录结构

```text
.
├── Makefile                # 编译自动化脚本
├── README.md               # 项目说明文档
├── main.c                  # 程序入口，主流程控制
├── lrc.c / lrc.h           # 歌词解析、链表维护与歌词显示逻辑
├── start_mplayer.c / .h   # 负责启动、控制 mplayer 子进程
├── console                  # 终端控制组件
│   ├── console.c           # 终端光标移动、颜色修改、清屏等工具函数
│   └── console.h
├── song                    # 存放 MP3 音频文件的目录
│   └── 鲜花.mp3
└── lrc                     # 存放 LRC 歌词文件的目录
    └── 鲜花.lrc
```

---

## 🚀 功能特点

* **歌词精准同步**：解析 `.lrc` 文件中的时间标签，根据当前播放进度，实现歌词的动态高亮与滚动。
* **终端界面优化**：通过 `console` 模块封装的 ANSI 转义序列，实现终端界面的定点刷新，避免闪烁。
* **音频后台播放**：通过 `start_mplayer` 模块拉起外部播放器进程，实现音乐播放与歌词刷新的并发执行。
* **一键编译**：提供完整的 `Makefile`，极大简化了编译与清理的步骤。

## 🛠️ 环境准备

1. **操作系统**：Linux 或 macOS (推荐 Ubuntu/Debian)
2. **依赖工具**：

* 需要系统提前安装 `mplayer`。如果未安装，请使用包管理器进行安装：

```bash
# Ubuntu / Debian
sudo apt-get update
sudo apt-get install mplayer

* 安装核心编译工具（`gcc`, `make`）：
sudo apt-get install build-essential
```

## 💻 编译与运行

在项目根目录下打开终端，执行以下命令：

### 1. 编译项目

使用 `Makefile` 自动化编译生成可执行文件 `music_player`：

```bash
make
```

### 2. 运行播放器

运行编译出来的程序（确保 `song/` 和 `lrc/` 目录下有对应的歌曲和歌词）：

```bash
./play
```

### 3. 清理编译产物

如果你需要清除生成的 `.o` 文件或可执行文件，可以运行：

```bash
make clean
```

## 📝 核心模块简介

* **`main.c`**：初始化程序，调用 `start_mplayer` 播放音乐，并开启歌词同步主循环。
* **`lrc.c`**：负责读取并按行解析 `.lrc` 文件，将时间（分:秒:毫秒）转换为总毫秒数，并按时间顺序存储到**链表**中，方便后续遍历对比。
* **`start_mplayer.c`**：利用 `fork()` 和 `exec` 系列系统调用（或 `popen`），在子进程中拉起 `mplayer`。
* **`console.c`**：提供控制台的艺术化处理（如隐藏光标 `\033[?25l`、设置颜色 `\033[1;31m` 等），让终端界面更像一个真正的播放器。

### 💡 小贴士（你可以根据实际情况微调）

1. **编码问题**：如果终端显示歌词乱码，请确保你的 `.lrc` 文件编码格式（如 UTF-8）与终端的编码格式一致。
