Git 是目前全球最流行的**分布式版本控制系统**。简单来说，它就像是写文档时的“历史记录”和“自动备份”大师，能帮你记录代码的每一次修改，并在需要时轻松实现“时光倒流”。

以下是为你整理的 Git 基础操作指南，帮你快速上手。
## 1. 核心概念：Git 的三个工作区域

在开始敲命令之前，理解 Git 的工作流程至关重要。Git 的核心结构由三个区域组成：

- **工作区 (Working Directory)：** 你电脑上实际看到并修改的代码文件夹。
- **暂存区 (Staging Area/Index)：** 临时的“缓存区”。你在这里挑选准备提交的修改，就像把商品放进购物车。
- **本地仓库 (Local Repository)：** 存放所有历史版本的地方。一旦提交到这里，你的代码就有了永久的历史记录。

## 2. 起步与配置

在你第一次使用 Git 时，需要告诉系统你是谁。

Bash

```bash
# 设置全局用户名
git config --global user.name "你的名字"

# 设置全局邮箱
git config --global user.email "你的邮箱@example.com"

# 查看当前的所有配置
git config --list
```

### 初始化或克隆仓库

要开始进行版本控制，你有两种选择：

- **选择 A：在本地新建仓库**

  进入你的项目文件夹，运行以下命令将其变成 Git 仓库：

  

  ```bash
  git init
  ```

- **选择 B：从远程服务器克隆现有仓库**

  如果你想参与别人的项目，或者把云端（如 GitHub/Gitee）的代码下载到本地：

  

  ```bash
  git clone <远程仓库的URL>
  ```

## 3. 最常用的核心基础命令

这是你每天写代码时，使用频率最高的“四大金刚”：



```
# 1. 查看当前仓库的状态（哪些文件修改了、哪些放进了暂存区）
git status

# 2. 将文件的修改添加到“暂存区”
git add <文件名>       # 添加指定文件
git add .            # 添加当前目录下所有修改过的文件（最常用）

# 3. 将暂存区的内容提交到“本地仓库”，并附加一条提交日志
git commit -m "这次提交的说明信息（例如：修复了登录页面的 Bug）"

# 4. 查看提交的历史记录
git log
# 如果觉得信息太多，可以用下面这个精简版
git log --oneline
```

> **💡 记忆小贴士：** > 每天的标配流程：修改代码 ----> `git status`（看一眼）----> `git add .`（打包） ---->`git commit -m "..."`（存盘）。

## 4. 分支操作（Branch）

分支是 Git 最强大的功能之一。它允许你在不影响主线代码（通常是 `main` 或 `master`）的情况下，安全地开发新功能或修复 Bug。

Bash

```bash
# 查看所有本地分支（当前所在分支前面会有 * 号）
git branch

# 创建一个新分支
git branch <新分支名称>

# 切换到指定分支
git switch <分支名称>   # 或者使用老命令：git checkout <分支名称>

# 【快捷命令】创建并直接切换到新分支
git switch -c <新分支名称>

# 合并分支（先切换到主分支，然后把新功能分支合并进来）
git switch main
git merge <新功能分支名称>

# 删除已经没用的分支
git branch -d <分支名称>
```

## 5. 远程仓库交互（Push & Pull）

当你的代码需要同步到 GitHub、Gitee 或公司内部的 GitLab 时，就会用到远程操作。

Bash

```
# 查看当前配置的远程仓库服务器
git remote -v

# 关联一个远程仓库（如果是在本地 git init 的项目）
git remote add origin <远程仓库的URL>

# 将本地的分支推送到远程仓库（第一次推送主分支时加 -u，以后直接 git push 即可）
git push -u origin main

# 从远程仓库拉取最新的代码，并自动合并到你的本地分支
git pull

# 获取远程仓库的最新动态，但不自动合并（比 pull 更安全，供查看用）
git fetch
```

## 6. 撤销与后悔药

写代码难免犯错，Git 提供了强大的撤销机制：

- **情况一：改乱了工作区，想放弃某个文件的修改，恢复到上一次提交的状态：**

  Bash

  ```bash
  git checkout -- <文件名>
  ```

- **情况二：不小心执行了 `git add`，想把文件从暂存区拿出来，但保留修改：**

  Bash

  ```bash
  git reset HEAD <文件名>
  ```

- **情况三：代码已经 `commit` 了，想撤销这次提交（版本回退）：**

  Bash

  ```bash
  # 回退到上一个版本（保留写好的代码，只是撤销 commit）
  git reset --soft HEAD^
  
  # 彻底回退到上一个版本（注意：这会抹去你上次 commit 之后的所有修改，慎用！）
  git reset --hard HEAD^
  ```
